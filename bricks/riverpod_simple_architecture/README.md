# riverpod_simple_architecture

[![Powered by Mason](https://img.shields.io/endpoint?url=https%3A%2F%2Ftinyurl.com%2Fmason-badge)](https://github.com/felangel/mason)

A opinionated riverpod simple architecture with built in features for production ready flutter apps.

_Generated by [mason][1] 🧱_



## This brick use a feature first approach, you can check the [architecture documentation](https://github.com/Shreemanarjun/river_bricks/blob/master/bricks/riverpod_simple_architecture/__brick__/architecture/architecture.md) for more details.

## 🧱 Scaffold contract (v3.1)

This brick is the **Phase-3 scaffold** for AppStudio-generated apps: it emits the
FOUNDATION ONLY. Feature content, provider integrations, and backend code are
agent-owned and implemented later from the app's spec bundle.

### Variables

| Var | Type | Purpose |
| --- | --- | --- |
| `project_name`, `app_title`, `org`, `app_description`, `author` | string | App identity (pubspec name, applicationId/bundle id, display name) |
| `seed_color` | string | Brand seed (6-hex, no `#`) — seeds `core/theme/brand_palette.dart` |
| `support_email`, `privacy_url`, `terms_url`, `refund_url` | string | Per-flavor `AppEnv` values (release-checklist gated when empty) |
| `api_base_url_development/staging/production` | string | Per-flavor API base URL; empty ⇒ `dioProvider` fails fast until the backend is wired |
| `backend` | enum `firebase\|supabase\|none` | Drives `.gitignore` firebase rules + analyzer excludes and setup tooling expectations |
| `languages` | list | Locale codes; `en` is always included and always the slang base. post_gen fans out `<code>.i18n.json` placeholder copies and regenerates strings |
| `features` | list | Feature skeletons (`"name"` or `{"name":"auth","data":true}`): folder + barrel + mirrored `test/features/<name>/` dir. CONTENT stays agent-owned |
| `routes` | list | `{name, path, requires_onboarding, requires_auth, initial}` per app screen. Splash (initial), `/settings`, and the `*` NotFound wildcard are brick-owned and added automatically. Guarded route sets REQUIRE an unguarded onboarding entry route and/or a `login`-named route (pre_gen fails loudly otherwise) |
| `providers` | list | Selected provider ids — drive the generated `.env.example` blocks |
| `responsive` | boolean | Include responsive_framework |

Flavors are a fixed convention: `{development, staging, production}`
(`AppFlavor`, `main_<flavor>.dart`, per-flavor `AppEnv`).

### Brick-owned file enumeration

**`MANIFEST.txt` (next to `brick.yaml`) enumerates every file this brick
owns** — emitted templates, post_gen codegen outputs, and the var-driven
per-language/per-feature additions, plus the android/ios files post_gen
patches (applicationId, android:label, `enableOnBackInvokedCallback`,
iOS bundle id, `CFBundleDisplayName`). Regenerating with the same vars
reproduces every file on that list; agents may only extend at marked seams
(`FIREBASE_INIT`, `AGENT-SEAM`, `guard_state.dart` re-pointing,
StorageKeys/PersistedStateKeys appends).

### What this brick must NEVER emit (agent-owned)

Feature page/controller/data content; `core/notifications/`,
`core/remote_config/`, `core/sync/`; attribution/analytics SDK backends or
event registries; Firebase crash-reporter wiring (use `tool/setup_firebase.sh`);
`shared/audio/`; brand-specific widgets or per-feature color extensions;
`firebase/`/`supabase/` backend projects; goals/TASKS/spec docs (bundle-owned);
re-pointing `guard_state.dart` at real feature state.

## Crash reporting (Firebase Crashlytics) 🐞

Apps ship with a no-op crash reporter, so they build and run with **no Firebase** — uncaught
errors are still logged via talker. To send crashes to Firebase Crashlytics:

```sh
dart pub global activate flutterfire_cli   # one-time
bash tool/setup_firebase.sh                 # run in your generated app
```

The script runs `flutterfire configure` (the one interactive step — log in and pick/create a
Firebase project), adds the Firebase deps, installs the Firebase crash reporter, wires it in, and
patches the Android Gradle plugins. iOS needs dSYM upload (see the Firebase docs) for symbolicated
crashes.

## Requirements ⚙️

- **Flutter** ≥ 3.38.0 / **Dart** ≥ 3.10.8 (required by flex_color_scheme 8 and multiple_result).
- **Android** `minSdk` ≥ **23** — `flutter_secure_storage` 10.x uses Android Keystore (AES‑GCM) and the encrypted Hive box relies on it. Set this in `android/app/build.gradle(.kts)`.
- **iOS** ≥ 12.

> All dependencies are pinned in the generated `pubspec.yaml`. Localization is generated with `dart run slang` and routes with `dart run build_runner build` (both run automatically by the post‑generation hook).

## Getting Started 🚀

### Install using Mason

```sh
dart pub global activate mason_cli

```
### Install brick from https://brickhub.dev/
```sh
 mason add -g riverpod_simple_architecture

```

### Install brick from github
```sh
 mason add -g riverpod_simple_architecture --git-url https://github.com/Shreemanarjun/river_bricks --git-path bricks/riverpod_simple_architecture

```
### Upgrade to newer version

```sh
mason upgrade --global

```

## Variables ✨

| Variable          | Description                                  | Default       | Type    |
| ----------------- | -------------------------------------------- | ------------- | ------- |
| `project_name`    | Project / package name                       | *(prompted)*  | string  |
| `app_title`       | Display title (launcher + in-app)            | derived       | string  |
| `seed_color`      | Brand seed color, **6-hex without `#`**      | `3F51B5`      | string  |
| `org`             | Reverse-domain org (→ `applicationId`)       | `com.example` | string  |
| `app_description` | Short description (pubspec + About)          | `""`          | string  |
| `author`          | Author / publisher                           | `""`          | string  |
| `support_email`   | Support email (About)                        | `""`          | string  |
| `privacy_url`     | Privacy policy URL (About)                   | `""`          | string  |
| `responsive`      | Include `responsive_framework`               | `false`       | boolean |

### Example

```sh
mason make riverpod_simple_architecture \
  --project_name unit_converter \
  --app_title "Unit Converter" \
  --seed_color 3F51B5 \
  --org dev.shreeman \
  --responsive false
```

App identity is centralised in `lib/const/app_config.dart` (`AppConfig`). The theme is
derived from `seed_color`. With `--responsive true`, `responsive_framework` and its
wrapper are included; otherwise they are omitted.

### Make using (with prompt)

```sh
mason make riverpod_simple_architecture

```
## OR (without prompt as an argument)

```sh
mason make riverpod_simple_architecture --project_name "Your Project Name"

```
